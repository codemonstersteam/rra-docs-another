# 0003 — Проектный конфиг во внешнем YAML (первая Go-зависимость)

## Статус

accepted

## Контекст

`--config <файл>` заложен в контракте (`api-specification/cli.md`) как проектный
конфиг: словари заголовков L4, профиль линтеров L2, пороги. Формат не был
зафиксирован. Со слайсом S5 (`fitness`, L5) появилась потребность вынести наружу
и **подключение к LLM**, и — главное — **промпты четырёх JTBD-ролей**, чтобы их
можно было дорабатывать без пересборки бинаря.

Проект до сих пор держал инвариант «zero-dependency / stdlib-онли» (`go.mod` без
`require`, «CLI без cobra», формулы L1 нативно). В Go нет stdlib-парсера YAML.
Развилка: JSON (stdlib, ноль зависимостей, но многострочные промпты в нём — боль)
против YAML (читаемые многострочные блоки `|`, но первая внешняя зависимость).

## Решение

1. **Формат `--config` — YAML.** Берём `gopkg.in/yaml.v3` как первую и (пока)
   единственную Go-зависимость проекта. Чистый Go, без CGo, де-факто стандарт.
   Инвариант «без cobra / логика нативно» сохраняется — зависимость только на
   парсинг конфига, не на доменную логику.
2. **Один файл с секциями**, не зоопарк. Дефолт вшит через `go:embed` — тул
   работает из коробки; `--config` переопределяет, флаги `--llm-*` переопределяют
   файл (приоритет: флаг > файл > вшитый дефолт). Битый файл → `config_invalid`.
3. **Секретов в YAML нет.** Ключ API читается из env. В YAML указывается лишь
   **имя** env-переменной (`api_key_env`), что делает её конфигурируемой для
   openai-совместимых эндпоинтов (OpenRouter/Azure/локальные). Отсутствие ключа в
   указанной env на команде, которой он нужен → `llm_unavailable`, код `2`.

Форма (дефолт через `go:embed`):

```yaml
llm:
  provider: anthropic           # | openai
  base_url: ""                  # для openai-совместимых
  model: claude-sonnet-4-6
  api_key_env: ANTHROPIC_API_KEY # ИМЯ env-переменной, не сам ключ
  budgets: { maintainer: docs_all, consumer: 2000, manager: 1500, agent: 8000 }
prompts:                        # то, что дорабатываем без пересборки
  maintainer: |
    …
  consumer: |
    …
  manager: |
    …
  agent: |
    …
```

## Последствия

- Загрузчик `--config` — **общая инфраструктура** (его ждут L4-словари, L2-профиль,
  пороги, теперь LLM-подключение и промпты), а не часть слайса. Заводится в
  `internal/cli` на бутстрапе, парсится в value-конфиг, инжектится как value-config
  в конструкторы (`NewLLMClient(conn)`) и в `Deps` слайса (промпты) — паттерн «как
  профиль линтера».
- Чтение файла — I/O на краю (`internal/cli`), не в адаптере и не в голове.
- Появляется `go.sum`; CI и `Dockerfile.runtime` уже делают `go mod download` —
  ломаться не должно.
- `api-specification/cli.md` дополняется: формат `--config` = YAML, схема секций,
  `api_key_env`.
